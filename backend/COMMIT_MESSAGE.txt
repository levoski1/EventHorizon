perf: integrate bullmq for background action processing

Add BullMQ-based queue system for reliable background processing of trigger
actions (Discord, Email, Telegram, Webhooks). The system gracefully falls
back to direct execution when Redis is unavailable.

Features:
- Guaranteed delivery with 3 automatic retries and exponential backoff
- Concurrent action execution (5 workers by default)
- Rate limiting (10 jobs/second) to prevent API throttling
- Job monitoring via /api/queue/* endpoints
- Graceful fallback to direct execution without Redis
- Comprehensive documentation and migration guides

Changes:
- Add BullMQ worker for background job processing
- Decouple event poller from HTTP calls for better reliability
- Add queue monitoring API endpoints (stats, jobs, retry, clean)
- Implement automatic retries with exponential backoff
- Add concurrency control and rate limiting
- Include graceful fallback when Redis is unavailable
